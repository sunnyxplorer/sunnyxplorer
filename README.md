From coding on my mom's phone in high school to creating tools I use every day, coding has been more than a skill—it’s a life-long pursuit of building and optimizing systems. I started with HTML and CSS, dove deeper with TypeScript and Node.js, and now I’m honing my expertise in Rust, where I’ve discovered a new world of high-performance, low-level programming that’s both exhilarating and humbling. My love-hate relationship with Rust stems from its uncompromising standards and powerful control over memory, lifetimes, and concurrency. Unlike Node.js, which I admire for its simplicity and the vast NPM ecosystem, Rust speaks directly to the machine. Rust’s design gives me the thrill of producing pure, lean executables that need no engine like V8—just efficient, directly-compiled code that the CPU understands. Learning Rust is a deep dive into the core of computer science, from raw memory management to system interrupts, concepts that TypeScript and JavaScript rarely explore. Driven by the magic of operating systems, kernels, and the raw mechanics of memory, I’m continually fascinated by how software can directly communicate with hardware. As I transition more into Rust, I’m embracing a new frontier of systems programming, where every line of code is an exercise in precision and power. I’m passionate about coding in ways that stretch my limits—whether the challenge is improving latency, minimizing overhead, or designing for concurrency. Coding is something I’d do even if I wasn’t paid for it, as it represents both my craft and my playground.
